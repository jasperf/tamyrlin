Understanding Movement

FUNDAMENTALS

Position vs. Change in Position

Position is where something is located (x, y coordinates).

Change in position (or velocity) is how much the position changes each frame.

# Position - where the player is NOW player_x = 100 player_y = 50 # Velocity - how much position changes each frame change_x = 5 # Move right 5 pixels per frame change_y = 0 # No vertical movement # Update position each frame player_x += change_x player_y += change_y

Key insight: We don't set position directly. We change it by adding velocity each frame.

Keyboard Input in Arcade

Arcade provides methods to capture keyboard input:

def on_key_press(self, key, modifiers): # Called when a key is pressed down if key == arcade.key.RIGHT: self.change_x = 5 elif key == arcade.key.LEFT: self.change_x = -5 def on_key_release(self, key, modifiers): # Called when a key is released if key == arcade.key.RIGHT or key == arcade.key.LEFT: self.change_x = 0

Important: Always set velocity back to 0 when keys are released, or your player will keep moving!

The Update Method

The on_update() method is called every frame (typically 60 times per second). This is where you update positions:

def on_update(self, delta_time): # delta_time is time since last frame in seconds self.player.center_x += self.player.change_x self.player.center_y += self.player.change_y

Note: For sprites, use center_x and center_y. For simple positions, use regular variables.

Screen Boundaries

Prevent your player from going off-screen by checking boundaries:

def on_update(self, delta_time): self.player.center_x += self.player.change_x self.player.center_y += self.player.change_y # Keep player on screen if self.player.left < 0: self.player.left = 0 if self.player.right > self.width: self.player.right = self.width if self.player.bottom < 0: self.player.bottom = 0 if self.player.top > self.height: self.player.top = self.height

Sprite properties: left, right, top, bottom are automatically calculated from center_x, center_y, and the sprite's width/height.

HANDS-ON CODING

Add Keyboard Movement to Your Game

EXTEND YOUR SPRITES SHEET PROJECT

Build on your game from the Sprites & Graphics sheet. Add keyboard controls to move your player sprite smoothly across the screen.

Starter Code

Add these methods to your MyGame class:

def __init__(self): super().__init__(800, 600, "Movement Demo") arcade.set_background_color(arcade.csscolor.DARK_BLUE) self.player = arcade.Sprite("assets/sprites/player.png", 0.5) self.player.center_x = 100 self.player.center_y = 100 self.player.change_x = 0 self.player.change_y = 0 def on_key_press(self, key, modifiers): if key == arcade.key.RIGHT: self.player.change_x = 5 elif key == arcade.key.LEFT: self.player.change_x = -5 elif key == arcade.key.UP: self.player.change_y = 5 elif key == arcade.key.DOWN: self.player.change_y = -5 def on_key_release(self, key, modifiers): if key == arcade.key.RIGHT or key == arcade.key.LEFT: self.player.change_x = 0 elif key == arcade.key.UP or key == arcade.key.DOWN: self.player.change_y = 0 def on_update(self, delta_time): self.player.center_x += self.player.change_x self.player.center_y += self.player.change_y # Keep player on screen if self.player.left < 0: self.player.left = 0 if self.player.right > self.width: self.player.right = self.width if self.player.bottom < 0: self.player.bottom = 0 if self.player.top > self.height: self.player.top = self.height

Remember: Don't forget to add self.player_list.append(self.player) in your setup() method and draw the sprite list in on_draw().

Checkpoints

  • Player sprite appears on screen at position (100, 100)
  • Pressing arrow keys moves the player smoothly
  • Releasing keys stops the player
  • Player cannot move off-screen (boundaries work)

Troubleshooting

  • Player doesn't move: Check that you're using change_x and change_y, not setting center_x directly. Also verify on_key_press and on_key_release are registered.
  • Player disappears: Make sure you're drawing the player sprite list in on_draw().
  • Player slides off screen: Check your boundary checks - use left, right, top, bottom properties.
  • Keys stick: Ensure on_key_release sets velocity back to 0.
