Coins & Scoring

CORE CONCEPTS

Collectible Items

Collectibles are sprites that the player can pick up by touching them. When collected, the item disappears and something happens — the score increases, a sound plays, or the player gains a power-up.

The pattern:

  1. Create a Coin sprite class that extends arcade.Sprite
  2. Store coins in a SpriteList so Arcade can draw and manage them
  3. Each frame, check for collisions between the player and the coin list
  4. When a collision is found, remove the coin and update the score

The Coin Class

A coin is just a sprite with a position. You can use any image — a gold circle, a gem, a star:

class Coin(arcade.Sprite): def __init__(self, x, y): super().__init__( "assets/sprites/coin.png", COIN_SCALE ) self.center_x = x self.center_y = y

Tip: Place coins above platforms so the player can reach them by jumping.

SpriteList for Coins

Just like platforms, coins go into their own SpriteList. This lets Arcade draw them all at once and check collisions efficiently:

# In Game.setup() self.coin_list = arcade.SpriteList() # Add coins at specific positions coin_positions = [(200, 150), (350, 200), (500, 250)] for x, y in coin_positions: coin = Coin(x, y) self.coin_list.append(coin)

Collision Detection & Collection

Use check_for_collision_with_list() to detect when the player touches a coin. This returns a list of all coins the player is currently overlapping:

# In Game.on_update() coins_hit = arcade.check_for_collision_with_list( self.player, self.coin_list ) for coin in coins_hit: coin.remove_from_sprite_lists() self.score += 1

Key method: remove_from_sprite_lists() removes the coin from every SpriteList it belongs to. After removal, it won't be drawn or included in future collision checks.

Displaying the Score

Draw the score on screen using arcade.draw_text(). Use a separate UI camera so the score stays in a fixed position even when the main camera follows the player:

# In Game.on_draw() self.camera.use() # Main camera for game world self.platform_list.draw() self.coin_list.draw() self.player_list.draw() self.ui_camera.use() # UI camera for fixed elements arcade.draw_text( f"Score: {self.score}", 10, self.height - 30, arcade.color.WHITE, 18 )

Sound Effects

Sound makes collection feel satisfying. Load sounds once in setup() and play them on collection:

# In Game.setup() self.coin_sound = arcade.load_sound("assets/sounds/coin.wav") # In Game.on_update(), when coin collected arcade.play_sound(self.coin_sound)

Why load once? Loading a sound file from disk is slow. By loading it once and storing the reference, play_sound() is instant.

⚠️

Common Pitfall: If you call arcade.load_sound() inside on_update(), it will reload the file every frame (60 times per second!), causing lag. Always load sounds in setup().

Project Task

BUILD YOUR GAME

Add Coins & Scoring to Your Game

Time to reward your player for exploring! Add collectible coins, a score display, and a satisfying collection sound.

Step 1: Create the Coin Class

Create a new file coin.py in your project (or add the class to your existing game.py):

import arcade COIN_SCALE = 0.5 class Coin(arcade.Sprite): def __init__(self, x, y): super().__init__( "assets/sprites/coin.png", COIN_SCALE ) self.center_x = x self.center_y = y

No coin image yet? You can use arcade.make_circle_texture(32, arcade.color.GOLD) to generate a simple circle, or download free sprites from kenney.nl.

Step 2: Add Coins & Score in setup()

In your Game class, create the coin list, add coins, initialise the score, and load the collection sound:

def setup(self): # ... existing setup code ... # Score self.score = 0 # Sound self.coin_sound = arcade.load_sound( "assets/sounds/coin.wav" ) # Create coin list self.coin_list = arcade.SpriteList() # Place coins above platforms coin_positions = [ (200, 150), (300, 150), (450, 200), (550, 250), (700, 300), ] for x, y in coin_positions: coin = Coin(x, y) self.coin_list.append(coin)

Step 3: Detect & Handle Collection in on_update()

Add coin collision detection to your on_update method, after updating player movement:

def on_update(self, delta_time): # ... existing movement/gravity/platform code ... # Check for coin collection coins_hit = arcade.check_for_collision_with_list( self.player, self.coin_list ) for coin in coins_hit: coin.remove_from_sprite_lists() self.score += 1 arcade.play_sound(self.coin_sound)

Step 4: Draw Coins & Score in on_draw()

Draw the coins with the main camera and the score with the UI camera:

def on_draw(self): self.clear() # Game world (follows player) self.camera.use() self.platform_list.draw() self.coin_list.draw() self.player_list.draw() # UI (fixed on screen) self.ui_camera.use() arcade.draw_text( f"Score: {self.score}", 10, self.height - 30, arcade.color.WHITE, 18 )

Step 5: Add a Coin Image

If you don't have a coin sprite yet, you can create one programmatically:

# Option 1: Generate a simple circle texture class Coin(arcade.Sprite): def __init__(self, x, y): super().__init__() self.texture = arcade.make_circle_texture( 32, arcade.color.GOLD ) self.center_x = x self.center_y = y

Or download free game assets from kenney.nl and place the image in assets/sprites/.
