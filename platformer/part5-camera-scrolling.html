Camera & Viewport

CORE CONCEPTS

What Is a Camera?

In game development, the camera defines what portion of the game world is visible on screen. Think of it like a movie camera - it can move around, zoom in/out, and focus on different parts of the scene.

Why we need it:

  • Large worlds: Your game world is likely bigger than the screen. The camera shows a viewport into that world.
  • Follow player: The camera should follow the player so they're always visible.
  • Control boundaries: The camera shouldn't show empty space outside your level.

In Arcade: The arcade.Camera class makes this easy!

import arcade self.camera = arcade.Camera(self.width, self.height)

How Cameras Work in Arcade

Arcade's camera system has a few key concepts:

  • Camera position: The center point of what the camera is showing (camera.position = (x, y))
  • Viewport: The visible area on screen (same size as your window)
  • Activating the camera: You must call camera.use() before drawing
  • Sprite lists: Each SpriteList can use a different camera for parallax effects

Basic usage pattern:

# In setup() self.camera = arcade.Camera(SCREEN_WIDTH, SCREEN_HEIGHT) # In on_draw() self.camera.use() # Activate the camera self.player_list.draw() # Draw sprites

Following the Player

To make the camera follow the player, simply update the camera's position to match the player's position in the on_update method:

def on_update(self, delta_time): # Update player position self.player_list.update() # Position camera at player self.camera.position = (self.player.center_x, self.player.center_y)

But wait! This centers the player perfectly, which means the player can only move half-screen from the edge. We can do better with viewport margins...

Viewport Margins & Smooth Scrolling

For a better experience, use move_to_with_speed() which allows viewport margins and smooth movement:

# Constants CAMERA_SPEED = 0.5 # In on_update() self.camera.move_to_with_speed( self.player.center_x, self.player.center_y, CAMERA_SPEED)

How it works: move_to_with_speed() creates smooth camera movement. The speed parameter (0.1-1.0) controls how quickly the camera catches up to the player. Lower = smoother but slower.

Level Boundaries

Your camera shouldn't show empty space outside your level. Constrain the camera to stay within the level boundaries:

# Clamp camera to level boundaries left_limit = self.width / 2 right_limit = LEVEL_WIDTH - self.width / 2 camera_x = max(left_limit, min(self.player.center_x, right_limit)) camera_y = max(self.height/2, min(self.player.center_y, LEVEL_HEIGHT - self.height/2)) self.camera.move_to_with_speed(camera_x, camera_y, CAMERA_SPEED)

Pro tip: Set LEVEL_WIDTH and LEVEL_HEIGHT to match your actual level dimensions.

Multiple Cameras for Advanced Effects

For parallax scrolling (background moving slower than foreground) or fixed UI, use multiple cameras:

# Create cameras for different layers self.main_camera = arcade.Camera(self.width, self.height) self.ui_camera = arcade.Camera(self.width, self.height) # Assign to sprite lists self.player_list = arcade.SpriteList(camera=self.main_camera) self.ui_list = arcade.SpriteList(camera=self.ui_camera)
⚠️

Common Pitfall: Forgetting to call camera.use() before drawing will result in nothing appearing on screen! Also, position your camera after updating player position, not before.

Complete Camera Flow

The complete camera implementation flow:

  1. Create camera: In setup(), create your camera object
  2. Position camera: In on_update(), update camera position to follow player
  3. Clamp to boundaries: Ensure camera stays within level limits
  4. Activate camera: In on_draw(), call camera.use() before drawing
  5. Draw sprites: All sprites assigned to that camera will be drawn correctly

Pro tip: For the best feel, use move_to_with_speed() with a speed between 0.3 and 0.8.

BUILD YOUR GAME

Add Camera Scrolling to Your Game

Now it's time to implement camera scrolling! Make your camera follow the player as they move through the level.

Step 1: Create the Camera

In your Game class setup() method, create a camera and assign it to your sprite lists:

# Create the camera self.camera = arcade.Camera(self.width, self.height) # Assign camera to sprite lists self.player_list = arcade.SpriteList(camera=self.camera) self.platform_list = arcade.SpriteList(camera=self.camera)

Step 2: Define Level Boundaries

Add level boundary constants to your Game class. These define how far the camera can scroll:

# Level dimensions (adjust to your level) LEVEL_WIDTH = 2000 # Total width in pixels LEVEL_HEIGHT = 800 # Total height in pixels

Step 3: Position the Camera in on_update

Update the camera position in your on_update method to follow the player with smooth movement and boundary clamping:

def on_update(self, delta_time): # Update sprite positions self.player_list.update() self.platform_list.update() # Camera speed (0.1-1.0) CAMERA_SPEED = 0.5 # Clamp camera to level boundaries camera_x = max(self.width/2, min(self.player.center_x, LEVEL_WIDTH - self.width/2)) camera_y = max(self.height/2, min(self.player.center_y, LEVEL_HEIGHT - self.height/2)) # Move camera smoothly self.camera.move_to_with_speed(camera_x, camera_y, CAMERA_SPEED)

How it works: move_to_with_speed() creates smooth movement. The min() and max() functions keep the camera within the level.

Step 4: Activate Camera in on_draw

In your on_draw method, activate the camera before drawing sprites:

def on_draw(self): self.clear() # Activate camera for sprite drawing self.camera.use() # Draw sprite lists self.platform_list.draw() self.player_list.draw()

Important: You must call camera.use() before drawing any sprites that should be affected by the camera.

Step 5: Add UI Camera (Optional)

For score and other UI that should stay fixed on screen, create a separate UI camera:

# In setup() self.ui_camera = arcade.Camera(self.width, self.height) # In on_draw(), after drawing game elements self.ui_camera.use() arcade.draw_text(f"Score: {self.score}", 10, 10, arcade.color.WHITE)

Why: The UI camera is never moved, so UI elements stay in the same screen position.
