Levels & Goals

CORE CONCEPTS

From a Sandbox to a Game

So far your player can run, jump, collect coins, and dodge enemies — but there's nowhere to go. A game needs a goal (something to reach) and a win condition (what counts as success). Once one level can be won, the natural next step is having several levels.

The pattern:

  1. Add a goal sprite (a flag) at the end of the level
  2. Each frame, check whether the player has reached the goal
  3. Store each level's layout as data (positions of platforms, coins, enemies, goal)
  4. Write one setup_level() method that builds a level from that data
  5. On reaching the goal, advance to the next level — or win the game

The Goal Sprite

The goal is just a single sprite placed at the end of the level. Because there's only one, you store it on its own (not in a SpriteList):

# In Game.setup_level() self.goal = arcade.Sprite( "assets/sprites/flag.png", 0.5 ) self.goal.center_x = 1180 self.goal.center_y = 128

Detecting the Win

Because the goal is a single sprite, use arcade.check_for_collision() — the version that compares two sprites — rather than the ..._with_list() version:

# In Game.on_update() if arcade.check_for_collision(self.player, self.goal): self.advance_level()

Two collision helpers, two jobs:

  • check_for_collision(a, b) — one sprite vs one sprite (the goal)
  • check_for_collision_with_list(a, lst) — one sprite vs a whole list (coins, enemies)

Levels as Data

Hard-coding one level inside setup() works for a single screen, but it doesn't scale. Instead, describe each level as a dictionary and keep them in a list. Data is easy to add to, tweak, and even load from a file later:

LEVELS = [ { "platforms": [(200, 96), (400, 160)], "coins": [(400, 200)], "enemies": [(300, 96, 250, 500)], "goal": (760, 128), }, { "platforms": [(150, 140), (500, 220)], "coins": [(150, 180), (500, 260)], "enemies": [(350, 96, 300, 600)], "goal": (900, 128), }, ]

Building a Level from Data

One method reads a level dictionary and constructs every sprite list from it. The key idea: rebuild the lists fresh each time so the previous level's sprites don't linger:

def setup_level(self, level_index): data = LEVELS[level_index] # Fresh lists every time self.platform_list = arcade.SpriteList() self.coin_list = arcade.SpriteList() self.enemy_list = arcade.SpriteList() for x, y in data["coins"]: self.coin_list.append(Coin(x, y)) for x, y, left, right in data["enemies"]: self.enemy_list.append(Enemy(x, y, left, right)) gx, gy = data["goal"] self.goal = arcade.Sprite("assets/sprites/flag.png", 0.5) self.goal.center_x = gx self.goal.center_y = gy # Move the player back to the start self.player.center_x = START_X self.player.center_y = START_Y

Advancing & Winning

Keep an index of the current level. When the player reaches the goal, move to the next one — unless that was the last level, in which case the whole game is won:

def advance_level(self): self.current_level += 1 if self.current_level >= len(LEVELS): self.game_won = True else: self.setup_level(self.current_level)

Notice current_level >= len(LEVELS) rather than ==. Using >= is a safe habit — it still catches the end even if the index ever overshoots.

⚠️

Common Pitfall: If setup_level() appends to the existing lists instead of creating fresh SpriteList() objects, every level keeps the previous level's platforms, coins, and enemies. The screen fills up with leftovers. Always start each level with new, empty lists.

Project Task

BUILD YOUR GAME

Add a Goal and a Second Level

Turn your sandbox into a real game: add a goal flag, detect the win, and load levels from a data list. Build on the game you made in Part 7.

Step 1: Describe Your Levels as Data

Near the top of game.py, define a list of level dictionaries. Start with two:

START_X, START_Y = 64, 128 LEVELS = [ { "coins": [(200, 150), (400, 200)], "enemies": [(300, 96, 250, 500)], "goal": (760, 128), }, { "coins": [(150, 180), (500, 260)], "enemies": [(350, 96, 300, 600), (600, 96, 550, 760)], "goal": (880, 128), }, ]

Tip: Keep your platforms either in this data too, or in a shared ground layout — whichever you prefer. The important part is that coins, enemies, and the goal come from the level's data.

Step 2: Write setup_level()

Replace the level-specific parts of your old setup() with a method that builds a level from its data — with fresh lists each time:

def setup_level(self, level_index): data = LEVELS[level_index] self.coin_list = arcade.SpriteList() self.enemy_list = arcade.SpriteList() for x, y in data["coins"]: self.coin_list.append(Coin(x, y)) for x, y, left, right in data["enemies"]: self.enemy_list.append(Enemy(x, y, left, right)) gx, gy = data["goal"] self.goal = arcade.Sprite("assets/sprites/flag.png", 0.5) self.goal.center_x = gx self.goal.center_y = gy self.player.center_x = START_X self.player.center_y = START_Y

Step 3: Track State & Kick Off Level 1

In setup(), create the player once, then load the first level:

def setup(self): # Player, score, lives, platforms created here (once) # ... existing setup code ... self.current_level = 0 self.game_won = False self.setup_level(self.current_level)

Step 4: Detect the Goal & Advance

In on_update(), after handling movement and enemies, check the goal:

def on_update(self, delta_time): if self.game_won: return # ... existing movement / coins / enemies ... if arcade.check_for_collision(self.player, self.goal): self.advance_level() def advance_level(self): self.current_level += 1 if self.current_level >= len(LEVELS): self.game_won = True else: self.setup_level(self.current_level)

Step 5: Draw the Goal, Level Number & Win Screen

Draw the goal with the main camera, and the level number plus any win message with the UI camera:

def on_draw(self): self.clear() self.camera.use() self.platform_list.draw() self.coin_list.draw() self.enemy_list.draw() arcade.draw_sprite(self.goal) self.player_list.draw() self.ui_camera.use() arcade.draw_text( f"Level {self.current_level + 1} Score: {self.score}", 10, self.height - 30, arcade.color.WHITE, 18 ) if self.game_won: arcade.draw_text( "YOU WIN!", self.width / 2, self.height / 2, arcade.color.GREEN, 48, anchor_x="center" )
