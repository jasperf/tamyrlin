Enemies & Hazards

CORE CONCEPTS

What Makes an Enemy?

An enemy is a sprite that moves on its own and threatens the player. Unlike a coin (which the player wants to touch), an enemy is something the player must avoid — or defeat. The building blocks are exactly the ones you already know: a sprite, a SpriteList, and collision detection.

The pattern:

  1. Create an Enemy sprite class that extends arcade.Sprite
  2. Give each enemy a patrol — move it every frame and reverse at boundaries
  3. Store enemies in their own SpriteList
  4. Each frame, check for collisions between the player and the enemy list
  5. Decide the outcome: stomp from above defeats the enemy, side contact hurts the player

The Enemy Class & Patrolling

The trick to a patrolling enemy is giving it a left and right boundary. The enemy moves by change_x each frame; when it reaches an edge, you flip the sign of change_x so it walks the other way:

ENEMY_SCALE = 0.5 ENEMY_SPEED = 2 class Enemy(arcade.Sprite): def __init__(self, x, y, left, right): super().__init__( "assets/sprites/enemy.png", ENEMY_SCALE ) self.center_x = x self.center_y = y self.left_bound = left self.right_bound = right self.change_x = ENEMY_SPEED def update(self, delta_time=1/60): self.center_x += self.change_x if self.center_x < self.left_bound: self.change_x *= -1 elif self.center_x > self.right_bound: self.change_x *= -1

Arcade calls every sprite's update() method when you run self.enemy_list.update(). Putting the patrol logic inside the Enemy class keeps each enemy responsible for its own movement.

Building the Enemy List

Just like coins and platforms, enemies live in their own SpriteList. Each enemy gets a position and a patrol range:

# In Game.setup() self.enemy_list = arcade.SpriteList() # Enemy that patrols between x=300 and x=500 enemy = Enemy(300, 96, left=300, right=500) self.enemy_list.append(enemy)

Updating & Drawing Enemies

Call update() on the enemy list every frame so they patrol, and draw() them with the main (game-world) camera so they scroll with the level:

# In Game.on_update() self.enemy_list.update() # In Game.on_draw(), with the main camera active self.enemy_list.draw()

Stomp or Get Hurt

When the player collides with an enemy, the outcome depends on direction. In the classic platformer rule, if the player is falling and lands on top, the enemy is defeated. Otherwise, the player gets hurt:

# In Game.on_update() enemies_hit = arcade.check_for_collision_with_list( self.player, self.enemy_list ) for enemy in enemies_hit: if self.player.change_y < 0 and \ self.player.center_y > enemy.center_y: # Falling onto the enemy — defeat it enemy.remove_from_sprite_lists() self.player.change_y = 12 # little bounce self.score += 10 else: # Hit from the side — take damage self.hurt_player()

Why check change_y < 0? A negative vertical velocity means the player is moving downward. Combined with the player's centre being above the enemy's centre, that reliably means “landed on top.”

Lives & Respawning

Getting hurt should cost something. A simple approach: subtract a life, send the player back to the start, and end the game when lives run out:

def hurt_player(self): self.lives -= 1 if self.lives <= 0: self.game_over = True else: # Respawn at the start position self.player.center_x = START_X self.player.center_y = START_Y self.player.change_x = 0 self.player.change_y = 0
⚠️

Common Pitfall: If you forget the self.player.change_y < 0 check and simply defeat the enemy on any contact, the player can kill enemies by walking straight into them. The direction check is what makes the stomp feel fair.

Project Task

BUILD YOUR GAME

Add Patrolling Enemies to Your Game

Time to add danger! Create an enemy that marches back and forth, let the player stomp it from above, and lose a life on side contact. Build on the game you made in Part 6.

Step 1: Create the Enemy Class

Create a new file enemy.py (or add the class to your existing game.py):

import arcade ENEMY_SCALE = 0.5 ENEMY_SPEED = 2 class Enemy(arcade.Sprite): def __init__(self, x, y, left, right): super().__init__( "assets/sprites/enemy.png", ENEMY_SCALE ) self.center_x = x self.center_y = y self.left_bound = left self.right_bound = right self.change_x = ENEMY_SPEED def update(self, delta_time=1/60): self.center_x += self.change_x if self.center_x < self.left_bound or \ self.center_x > self.right_bound: self.change_x *= -1

No enemy image yet? Use arcade.make_circle_texture(32, arcade.color.RED) in place of the image path, or grab a free sprite from kenney.nl.

Step 2: Add Enemies & Lives in setup()

Create the enemy list, add a patrolling enemy or two, and give the player some lives:

def setup(self): # ... existing setup code ... # Lives & game state self.lives = 3 self.game_over = False # Enemy list self.enemy_list = arcade.SpriteList() self.enemy_list.append( Enemy(300, 96, left=250, right=500) ) self.enemy_list.append( Enemy(700, 96, left=650, right=850) )

Step 3: Patrol & Collisions in on_update()

Update the enemies, then resolve collisions with the player:

def on_update(self, delta_time): if self.game_over: return # ... existing movement / physics / coins ... self.enemy_list.update() enemies_hit = arcade.check_for_collision_with_list( self.player, self.enemy_list ) for enemy in enemies_hit: if self.player.change_y < 0 and \ self.player.center_y > enemy.center_y: enemy.remove_from_sprite_lists() self.player.change_y = 12 self.score += 10 else: self.hurt_player()

Step 4: The hurt_player() Helper

Add a method to the Game class that handles taking damage:

def hurt_player(self): self.lives -= 1 if self.lives <= 0: self.game_over = True else: self.player.center_x = 64 self.player.center_y = 128 self.player.change_x = 0 self.player.change_y = 0

Step 5: Draw Enemies, Lives & Game Over

Draw enemies with the main camera, then lives and any game-over text with the UI camera:

def on_draw(self): self.clear() self.camera.use() self.platform_list.draw() self.coin_list.draw() self.enemy_list.draw() self.player_list.draw() self.ui_camera.use() arcade.draw_text( f"Score: {self.score} Lives: {self.lives}", 10, self.height - 30, arcade.color.WHITE, 18 ) if self.game_over: arcade.draw_text( "GAME OVER", self.width / 2, self.height / 2, arcade.color.RED, 48, anchor_x="center" )
