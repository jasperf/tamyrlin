Gravity & Jumping

CORE CONCEPTS

What Is Gravity?

In platformer games, gravity is a constant downward force that pulls the player (and other objects) toward the bottom of the screen. Without gravity, your player would float in mid-air!

How we simulate it:

  • Gravity constant: A fixed value (e.g., 0.5) subtracted from the player's vertical velocity each frame
  • Vertical velocity: The change_y property determines how fast the player moves up or down
  • Positive vs negative: Positive change_y moves up, negative moves down

In code:

# Gravity constant - how fast the player falls GRAVITY = 0.5 # In the player's update method: self.change_y -= GRAVITY

Jump Mechanics

Jumping is the opposite of gravity - it gives the player an upward velocity. The key principles:

  • Jump velocity: A positive value (e.g., 12) applied to change_y
  • Instant velocity: Unlike gravity which accelerates, jumping is an instant force
  • Jump arc: Gravity immediately starts pulling the player down, creating a natural arc

Important: Players should only be able to jump when they're on the ground (standing on a platform), not in mid-air!

JUMP_VELOCITY = 12 def jump(self): if self.is_on_ground: self.change_y = JUMP_VELOCITY

Ground Detection

Before allowing a jump, we need to check if the player is standing on a platform. We do this by:

  1. Checking for collisions with platforms
  2. Verifying the player is moving downward (falling onto the platform)
  3. Setting is_on_ground = True when landing
  4. Setting is_on_ground = False when jumping or falling
# In on_update, after moving the player: hit_list = arcade.check_for_collision_with_list( self.player, self.platform_list ) self.player.is_on_ground = False for platform in hit_list: if self.player.change_y <= 0: # Player is falling onto platform self.player.change_y = 0 self.player.center_y = platform.top self.player.is_on_ground = True

Putting It All Together

The complete movement flow in the player's update() method:

  1. Apply gravity: Always pull the player down
  2. Handle keyboard input: Update change_x for left/right movement
  3. Move the player: Apply both change_x and change_y
  4. Check collisions: Detect if player is standing on a platform
  5. Handle jumps: Only allow jumps when on the ground

Pro tip: For smoother movement, apply gravity before checking for collisions!

⚠️

Common Pitfall: If your player gets stuck in a platform or falls through, check the order of operations. Gravity should be applied before collision detection, and the player should be pushed out of platforms they land on (by setting their center_y = platform.top).

Project Task

BUILD YOUR GAME

Add Gravity & Jumping to Your Player

Now it's time to implement the physics! Add gravity and jumping to your game project.

Step 1: Add Constants

At the top of your constants.py file (or in your game class), add these physics constants:

# Physics constants GRAVITY = 0.5 JUMP_VELOCITY = 12 PLAYER_MOVEMENT_SPEED = 5

Step 2: Add Ground Tracking to Player

In your Player class __init__, add a property to track if the player is on the ground:

class Player(arcade.Sprite): def __init__(self, x, y): super().__init__("assets/sprites/player.png", SCALE) self.center_x = x self.center_y = y self.is_on_ground = False

Step 3: Update Player Movement

Modify your Player class update() method to apply gravity. This gets called automatically by Arcade every frame if you add the player to a SpriteList and call player_list.update():

def update(self): # Apply gravity self.change_y -= GRAVITY # Move based on current velocity self.center_x += self.change_x self.center_y += self.change_y

Note: If your player doesn't have change_x and change_y yet, add them in __init__:

self.change_x = 0 self.change_y = 0

Step 4: Add Keyboard Jump Control

In your Game class, add a method to handle jump key presses. This should be in your window/view class:

def on_key_press(self, key, modifiers): if key == arcade.key.UP or key == arcade.key.SPACE: self.player.jump() elif key == arcade.key.LEFT: self.player.change_x = -PLAYER_MOVEMENT_SPEED elif key == arcade.key.RIGHT: self.player.change_x = PLAYER_MOVEMENT_SPEED def on_key_release(self, key, modifiers): if key == arcade.key.LEFT or key == arcade.key.RIGHT: self.player.change_x = 0

Step 5: Add Jump Method to Player

Add the jump method to your Player class. Make sure to only jump when on the ground:

def jump(self): if self.is_on_ground: self.change_y = JUMP_VELOCITY self.is_on_ground = False

Step 6: Handle Collisions with Platforms

In your Game class on_update method, add collision detection between the player and platforms:

def on_update(self, delta_time): # Move player sprites self.player_list.update() # Check for collision with platforms hit_list = arcade.check_for_collision_with_list( self.player, self.platform_list ) # Reset ground state self.player.is_on_ground = False for platform in hit_list: if self.player.change_y <= 0: # Player is falling onto platform self.player.change_y = 0 self.player.center_y = platform.top self.player.is_on_ground = True
