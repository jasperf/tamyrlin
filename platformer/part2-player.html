The Player Class

CORE CONCEPTS

Creating a Custom Player Class

The best practice is to create a Player class that extends arcade.Sprite. This gives you:

  • Inheritance: All sprite properties and methods (center_x, center_y, etc.)
  • Customization: Add your own properties (speed, health, etc.)
  • Organization: Keep player-specific code in one place
  • Reusability: Can create multiple player instances
class Player(arcade.Sprite): def __init__(self, x, y): # Call parent class init super().__init__("assets/sprites/player.png", SCALE) # Set starting position self.center_x = x self.center_y = y

Player Movement with Constants

Use constants for movement speed to make it easy to adjust:

# At the top of your file PLAYER_MOVEMENT_SPEED = 5 def on_key_press(self, key, modifiers): if key == arcade.key.RIGHT: self.player.change_x = PLAYER_MOVEMENT_SPEED elif key == arcade.key.LEFT: self.player.change_x = -PLAYER_MOVEMENT_SPEED

Why constants? If you decide the player should move faster, you only need to change one number!

Handling Keyboard Input

In your Game class, handle key presses and releases:

def on_key_press(self, key, modifiers): if key == arcade.key.RIGHT: self.player.change_x = PLAYER_MOVEMENT_SPEED elif key == arcade.key.LEFT: self.player.change_x = -PLAYER_MOVEMENT_SPEED elif key == arcade.key.UP: self.player.change_y = PLAYER_MOVEMENT_SPEED elif key == arcade.key.DOWN: self.player.change_y = -PLAYER_MOVEMENT_SPEED def on_key_release(self, key, modifiers): if key == arcade.key.RIGHT or key == arcade.key.LEFT: self.player.change_x = 0 elif key == arcade.key.UP or key == arcade.key.DOWN: self.player.change_y = 0

Player Update Method

Add an update() method to your Player class to handle position updates and boundaries:

class Player(arcade.Sprite): def update(self): # Move based on velocity self.center_x += self.change_x self.center_y += self.change_y

Note: Arcade automatically calls update() on all sprites in a SpriteList if you call sprite_list.update().

Why Use change_x and change_y?

Arcade sprites have built-in change_x and change_y properties:

  • Automatic movement: When you add a sprite to a SpriteList and call update(), positions are updated automatically
  • Consistency: Standard way to handle velocity in Arcade
  • Built-in: These properties exist on all Arcade sprites

Alternative: You can also manually update center_x and center_y in your Game's on_update() method.

Project Task

BUILD YOUR PLAYER

Create the Player Class

BUILD ON PART 1 PROJECT

Following Part 1 where you set up your project structure, now create a proper Player class with keyboard controls.

File: player.py

Create a new file called player.py in your mario-game/ folder:

import arcade SCALE = 0.5 PLAYER_MOVEMENT_SPEED = 5 class Player(arcade.Sprite): def __init__(self, x, y): super().__init__("assets/sprites/player.png", SCALE) self.center_x = x self.center_y = y

File: game.py

Update your game.py to use the Player class:

from player import Player class MyGame(arcade.Window): def __init__(self): super().__init__(SCREEN_WIDTH, SCREEN_HEIGHT, "Mario Game") arcade.set_background_color(arcade.csscolor.SKY_BLUE) self.player_list = arcade.SpriteList() self.player = Player(100, 100) self.player_list.append(self.player) def setup(self): # Already set up in __init__ pass def on_draw(self): self.clear() self.player_list.draw() def on_update(self, delta_time): self.player_list.update() def on_key_press(self, key, modifiers): if key == arcade.key.RIGHT: self.player.change_x = PLAYER_MOVEMENT_SPEED elif key == arcade.key.LEFT: self.player.change_x = -PLAYER_MOVEMENT_SPEED elif key == arcade.key.UP: self.player.change_y = PLAYER_MOVEMENT_SPEED elif key == arcade.key.DOWN: self.player.change_y = -PLAYER_MOVEMENT_SPEED def on_key_release(self, key, modifiers): if key == arcade.key.RIGHT or key == arcade.key.LEFT: self.player.change_x = 0 elif key == arcade.key.UP or key == arcade.key.DOWN: self.player.change_y = 0

Note: Make sure your main.py creates a MyGame instance and calls arcade.run().

Checkpoints

  • Player class created in player.py
  • Game class uses Player from player.py
  • Player sprite appears on screen
  • Arrow keys move the player smoothly
  • Player stops when keys are released

Troubleshooting

  • ImportError: Make sure player.py is in the same folder as game.py. Use from player import Player.
  • Player doesn't move: Check that on_key_press and on_key_release are properly indented in the Game class.
  • Player jumps to position: Make sure you're setting change_x and change_y, not center_x and center_y directly.
  • Player sprite not found: Verify the path "assets/sprites/player.png" is correct and the file exists.
