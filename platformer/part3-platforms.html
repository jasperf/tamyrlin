Platforms & Collision

CORE CONCEPTS

What Is a Platform?

A platform is a solid surface the player can stand on. In Arcade, platforms are simply sprites — images positioned in the game world. The magic happens when we detect collisions between the player sprite and the platform sprites.

  • Platform sprite: An image (e.g. a grass or stone tile) placed at a fixed position
  • Hit box: An invisible rectangle around the sprite used for collision checks
  • Collision: When two hit boxes overlap, Arcade reports a collision

SpriteLists — Grouping Sprites

Arcade uses SpriteList to manage groups of related sprites efficiently:

# Create a SpriteList for platforms self.platform_list = arcade.SpriteList() # Create individual platform sprites platform = arcade.Sprite("assets/sprites/grass.png", TILE_SCALE) platform.center_x = 200 platform.center_y = 50 # Add to the list self.platform_list.append(platform)

Why SpriteLists?

  • Fast drawing: Arcade draws the whole list in one GPU call
  • Batch collision: Check player against all platforms at once
  • Easy updates: Call platform_list.update() to update every sprite

Collision Detection

Arcade has a built-in function to check if one sprite collides with any sprite in a list:

# Returns a list of sprites the player is touching hit_list = arcade.check_for_collision_with_list( self.player, self.platform_list )

This returns a Python list of every platform sprite the player is currently overlapping. If the player is not touching any platform, it returns an empty list [].

Landing on Platforms

When the player collides with a platform while falling, we need to:

  1. Stop downward movement: Set change_y = 0
  2. Snap to the top: Set player.bottom = platform.top
def on_update(self, delta_time): # Apply gravity self.player.change_y -= GRAVITY # Move the player self.player.center_x += self.player.change_x self.player.center_y += self.player.change_y # Check for collision with platforms hit_list = arcade.check_for_collision_with_list( self.player, self.platform_list ) for platform in hit_list: if self.player.change_y < 0: self.player.bottom = platform.top self.player.change_y = 0

Key properties: Every Arcade sprite has top, bottom, left, and right properties that give the edges of the sprite's hit box.

Building a Ground Row

To create the ground, place platform tiles side-by-side using a loop:

# Create a row of ground tiles for x in range(0, 800, 64): tile = arcade.Sprite("assets/sprites/grass.png", TILE_SCALE) tile.center_x = x tile.center_y = 32 self.platform_list.append(tile)

The range(0, 800, 64) creates tiles at x = 0, 64, 128, 192 ... 768 — covering the full window width with 64-pixel-wide tiles.

Project Task

BUILD YOUR PLATFORMS

Add Platforms and Collision Detection

BUILD ON PART 2 PROJECT

Following Part 2 where you created the Player class, now add platforms the player can stand on.

Step 1: Add Constants

Add these constants to the top of your game.py:

TILE_SCALE = 0.5 GRAVITY = 0.5

Step 2: Create Platforms in setup()

In your Game class, create a platform SpriteList and add ground tiles:

def setup(self): # Player (from Part 2) self.player_list = arcade.SpriteList() self.player = Player(100, 300) self.player_list.append(self.player) # Platforms self.platform_list = arcade.SpriteList() # Ground row for x in range(0, 800, 64): tile = arcade.Sprite("assets/sprites/grass.png", TILE_SCALE) tile.center_x = x tile.center_y = 32 self.platform_list.append(tile) # Floating platforms plat1 = arcade.Sprite("assets/sprites/grass.png", TILE_SCALE) plat1.center_x = 300 plat1.center_y = 200 self.platform_list.append(plat1) plat2 = arcade.Sprite("assets/sprites/grass.png", TILE_SCALE) plat2.center_x = 500 plat2.center_y = 350 self.platform_list.append(plat2)

Step 3: Draw Platforms

Update on_draw() to draw the platform list:

def on_draw(self): self.clear() self.platform_list.draw() self.player_list.draw()

Step 4: Gravity and Collision

Update on_update() with gravity and collision handling:

def on_update(self, delta_time): # Apply gravity self.player.change_y -= GRAVITY # Update player position self.player.center_x += self.player.change_x self.player.center_y += self.player.change_y # Check for collision with platforms hit_list = arcade.check_for_collision_with_list( self.player, self.platform_list ) for platform in hit_list: if self.player.change_y < 0: self.player.bottom = platform.top self.player.change_y = 0

Note: If you don't have sprite images yet, you can use Arcade's built-in shapes or download free tiles from kenney.nl.

Checkpoints

  • Platform SpriteList created in setup()
  • Ground tiles cover the bottom of the screen
  • At least two floating platforms at different heights
  • Player falls due to gravity
  • Player lands on platforms and stops falling
  • Player does not fall through platforms

Troubleshooting

  • Player falls through platforms: Make sure you set player.bottom = platform.top in addition to setting change_y = 0. Both are needed!
  • Platforms not visible: Check that you call self.platform_list.draw() in on_draw(). Draw platforms before the player so the player appears on top.
  • Player stuck inside platform: Make sure collision check happens after updating the player position, not before.
  • Image not found: Verify the path to your tile image. Try using ":resources:images/tiles/grassMid.png" to use Arcade's built-in assets.
