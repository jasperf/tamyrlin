Sound, Animation & States

CORE CONCEPTS

What "Polish" Means

Your game already works — the player runs, jumps, collects, dodges enemies, and beats levels. Polish is everything that makes it feel finished: sound you can hear, sprites that move, and a proper title screen instead of dropping straight into gameplay.

Three areas we'll add:

  1. Audio — looping background music plus the sound effects from Part 6
  2. Animation — swapping between several textures so the player has a walk cycle
  3. Game states — a menu, the gameplay, and a game-over/win screen

Background Music

You already used arcade.load_sound() and arcade.play_sound() for coins in Part 6. Music is the same — just played on a loop at a lower volume so it sits under the action:

# In setup(), load once self.music = arcade.load_sound("assets/sounds/music.mp3") # Start it looping, quietly self.music_player = arcade.play_sound( self.music, volume=0.3, loop=True )

play_sound() returns a player object. Keep a reference to it (self.music_player) so you can stop it later with self.music_player.pause() or arcade.stop_sound(self.music_player).

Animated Sprites: A List of Textures

An animation is just several images shown in sequence. Load the frames into a list, then swap the sprite's texture to the next one over time:

class Player(arcade.Sprite): def __init__(self): super().__init__() self.walk_textures = [ arcade.load_texture(f"assets/player/walk{i}.png") for i in range(4) ] self.cur_frame = 0 self.time_since_change = 0 self.texture = self.walk_textures[0]

Advancing the Animation on a Timer

If you changed the texture every frame, the animation would blur past at 60 fps. Instead, accumulate delta_time and only step forward after a set interval. The modulo operator (%) wraps the frame index back to 0:

def update_animation(self, delta_time): self.time_since_change += delta_time if self.time_since_change > 0.1: # 10 fps self.time_since_change = 0 self.cur_frame = (self.cur_frame + 1) % len(self.walk_textures) self.texture = self.walk_textures[self.cur_frame]

Call it each frame from the game's update: self.player.update_animation(delta_time). Arcade also provides AnimatedWalkingSprite helpers, but rolling your own makes the mechanic clear.

Game States

Right now the game starts the instant you run it. Real games have a menu first, then gameplay, then a game-over screen. The simplest way to manage this is a single state variable that on_update() and on_draw() branch on:

# State constants STATE_MENU = "menu" STATE_PLAYING = "playing" STATE_GAME_OVER = "game_over" # In setup() self.state = STATE_MENU def on_update(self, delta_time): if self.state != STATE_PLAYING: return # freeze the world on menu / game over # ... movement, enemies, coins, goal ...

Switching States with Input

Key presses move the player between states. From the menu, a key starts the game; from game over, a key restarts it:

def on_key_press(self, key, modifiers): if self.state == STATE_MENU: if key == arcade.key.SPACE: self.state = STATE_PLAYING elif self.state == STATE_GAME_OVER: if key == arcade.key.SPACE: self.setup() # restart from scratch
⚠️

Common Pitfall: Calling arcade.play_sound() for music inside on_update() starts a new overlapping track 60 times a second — a wall of noise. Start music once (in setup() or when entering the playing state), and keep the returned player to control it.

FINISH YOUR GAME

Polish Your Platformer

This is the finishing pass. Add music, animate the player, and wrap the game in a menu and game-over screen. Build on the game you made in Part 8.

Step 1: Add Game States

Define state constants and start in the menu. Branch your update and draw on the state:

STATE_MENU = "menu" STATE_PLAYING = "playing" STATE_GAME_OVER = "game_over" def setup(self): # ... existing setup ... self.state = STATE_MENU

Step 2: Draw Each Screen

In on_draw(), show the right thing for the current state:

def on_draw(self): self.clear() if self.state == STATE_MENU: arcade.draw_text("MY PLATFORMER", self.width/2, self.height/2+20, arcade.color.WHITE, 40, anchor_x="center") arcade.draw_text("Press SPACE to play", self.width/2, self.height/2-30, arcade.color.GRAY, 18, anchor_x="center") return # STATE_PLAYING / GAME_OVER: draw the world self.camera.use() self.platform_list.draw() # ... coins, enemies, goal, player ...

Step 3: Switch States on Key Press

Use on_key_press() to move between screens:

def on_key_press(self, key, modifiers): if self.state == STATE_MENU and key == arcade.key.SPACE: self.state = STATE_PLAYING return if self.state == STATE_GAME_OVER and key == arcade.key.SPACE: self.setup() return # normal movement keys only matter while playing if self.state == STATE_PLAYING: if key == arcade.key.LEFT: self.player.change_x = -PLAYER_SPEED # ... right / jump ...

Step 4: Animate the Player

Load walk frames in the player, advance them on a timer, and call update_animation() from the game's update (only while playing):

# In Player.update_animation(self, delta_time) self.time_since_change += delta_time if self.time_since_change > 0.1: self.time_since_change = 0 self.cur_frame = (self.cur_frame + 1) % len(self.walk_textures) self.texture = self.walk_textures[self.cur_frame] # In Game.on_update(), while playing self.player.update_animation(delta_time)

Step 5: Add Music

Load the music once and start it looping when the game begins:

# In setup() self.music = arcade.load_sound("assets/sounds/music.mp3") self.music_player = None # When entering STATE_PLAYING (e.g. in on_key_press) if self.music_player is None: self.music_player = arcade.play_sound( self.music, volume=0.3, loop=True )
