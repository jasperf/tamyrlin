Concepts: Project Organization

Why structure matters in game development

🎯 Why Organize Your Project?

As your platformer game grows, good organization makes it easier to:

  • Find and edit code — Know where everything is
  • Fix bugs — Isolate problems to specific files
  • Add features — Add new code without breaking existing code
  • Reuse code — Use classes and functions across multiple parts

Arcade encourages this through its View system and Sprite classes.

🏗️ Recommended Project Structure

Here's the folder structure we'll use for our platformer game:

mario-game/
├── main.py # Entry point - runs the game
├── game.py # Game class with main logic
├── assets/
│ ├── sprites/ # Player, enemies, coins, etc.
│ └── sounds/ # Jump, coin, death effects
└── levels/
├── level1.json
└── level2.json

Tip: Keep file names lowercase with underscores. This is the Python convention.

🎨 Arcade's View System

Arcade uses a View system to organize different screens:

  • MenuView — Start menu
  • GameView — Main gameplay
  • GameOverView — When player dies
  • PausedView — When game is paused

Each View is a separate class. We'll start with a simple GameView.

Project Task: Create Your Project Structure

Set up the foundation for your platformer game

🎮 Create the Project Structure

ON YOUR MAC

Follow these steps to create your platformer game project:

  1. Open Terminal (use Spotlight: Cmd+Space, type "Terminal")
  2. Navigate to your Documents folder: cd ~/Documents
  3. Create a new folder for your game: mkdir mario-game
  4. Go into the folder: cd mario-game
  5. Create the folder structure:
    mkdir assets mkdir assets/sprites mkdir assets/sounds mkdir levels
  6. Create empty files for each component:
    touch main.py touch game.py touch levels/level1.json touch levels/level2.json

📁 Verify Your Structure

Run this command to see your structure:

tree # or on Mac: find . -type f -o -type d | head -20

You should see: mario-game/, assets/, levels/, main.py, game.py

📝 Add Starter Code

IN main.py AND game.py

Step 1: Open main.py in your text editor and add:

import arcade from game import Game def main(): """ Main entry point """ window = arcade.Window(800, 600, "Mario Platformer") game = Game(window) window.show_view(game) arcade.run() if __name__ == "__main__": main()

Step 2: Open game.py and add the Game class:

import arcade class Game(arcade.View): """ Main game class """ def __init__(self, window): super().__init__(window) # Window dimensions self.width = window.width self.height = window.height self.title = window.title # These will be created in setup() self.player = None self.camera = None def setup(self): """ Set up the game here """ pass # We'll add this in Part 2 def on_draw(self, delta_time): """ Render the screen """ self.clear() # Drawing code will go here

⚠️ Troubleshooting

"ModuleNotFoundError: No module named 'arcade'"

Run: pip3 install arcade

"File not found" errors

Make sure you're running from the mario-game folder: cd ~/Documents/mario-game

"ImportError: cannot import name 'Game' from 'game'"

Make sure game.py has the Game class defined correctly.
