Concepts: Working with Sprites

Load, scale, and draw game graphics

🖼️ What is a Sprite?

A sprite is a 2D image or animation that is integrated into a larger scene. In game development:

  • Sprites are the visual representations of game objects (player, enemies, coins, platforms)
  • They can be static images (PNG files) or animated (sprite sheets)
  • Arcade's Sprite class handles loading, positioning, scaling, and drawing
  • Sprites have properties like center_x, center_y, scale, angle

Example: Your player character, a coin, or an enemy are all sprites.

📁 Loading Sprite Images

To load a sprite image in Arcade:

import arcade # Load a sprite from a file player = arcade.Sprite("assets/sprites/player.png", 0.5) # Parameters: # - "assets/sprites/player.png" : Path to the image file # - 0.5 : Scale factor (0.5 = 50% of original size)

Supported formats: PNG (recommended), JPG, GIF

Best practice: Use PNG with transparent backgrounds for sprites.

📏 Coordinate System Recap

Remember from the Arcade Basics sheet:

  • Origin (0, 0) is at the bottom-left corner
  • X increases to the right
  • Y increases upwards
  • A sprite's position is defined by its center_x and center_y

This means player.center_x = 100, player.center_y = 100 places the sprite 100 pixels from the left and 100 pixels up from the bottom.

🎯 Sprite Lists

For efficient rendering, group sprites into SpriteList objects:

self.player_list = arcade.SpriteList() self.coin_list = arcade.SpriteList() # Add sprites to lists player = arcade.Sprite("assets/sprites/player.png", 0.5) player.center_x = 100 player.center_y = 100 self.player_list.append(player) # Draw all sprites in a list self.player_list.draw()

Why use SpriteLists? More efficient rendering, built-in collision detection, easier management of groups of sprites.

Project Task: Load and Display a Sprite

Apply what you've learned to your game project

🎨 Add a Player Sprite to Your Game

IN YOUR mario-game PROJECT

Follow these steps to add a player sprite to your existing mario-game/ project:

  1. Create a sprites folder inside assets/ if it doesn't exist: mkdir -p assets/sprites
  2. Find a simple player sprite image (PNG with transparency) or use the sample below
  3. Save it as assets/sprites/player.png
  4. In game.py, add a SpriteList in setup():
    def setup(self): self.player_list = arcade.SpriteList() self.player = arcade.Sprite("assets/sprites/player.png", 0.5) self.player.center_x = 400 self.player.center_y = 100 self.player_list.append(self.player)
  5. In on_draw(), add: self.player_list.draw()
  6. Run your game with python3 main.py

🎮 Sample Sprite

If you don't have a sprite, you can use this simple base64-encoded 32x32 player sprite as a placeholder:

# Save this as assets/sprites/player.png # (Right-click the image below and save, or create your own) # For now, you can use any 32x32 PNG image # Free sprites available at: https://kenney.nl/assets

Tip: Use free game asset websites like kenney.nl for high-quality sprites.

⚠️ Troubleshooting

"File not found" error: Double-check the path to your sprite image. Make sure it's in assets/sprites/player.png.

Sprite not appearing: Make sure you're calling setup() and self.player_list.draw() in the right places.

Sprite is too big/small: Adjust the scale parameter: arcade.Sprite("player.png", 0.5) - try values between 0.1 and 2.0.

📝 Complete Game Class

UPDATE game.py

Here's what your game.py should look like after adding the sprite:

import arcade class Game(arcade.View): def __init__(self, window): super().__init__(window) self.width = window.width self.height = window.height self.player_list = None def setup(self): self.player_list = arcade.SpriteList() self.player = arcade.Sprite("assets/sprites/player.png", 0.5) self.player.center_x = 400 self.player.center_y = 100 self.player_list.append(self.player) def on_draw(self, delta_time): self.clear() self.player_list.draw()
